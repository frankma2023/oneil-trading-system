# 抛盘日

## 1. 抛盘日概述

**抛盘日**是指机构投资者系统性卖出导致的市场弱势交易日，表现为：**成交量放大 + 价格无力上涨（或下跌）**。它反映了卖方力量主导市场，买盘无法推动价格上涨。

核心是判断市场顶部和趋势反转，识别**机构抛售行为**而非散户恐慌。

**80%的情况只使用“标准抛盘日”**。特殊和盘中反转类，只在市场极端横盘/震荡期才辅助使用。


## 2. 抛盘日类型

### 2.1 标准抛盘日

**定义：**
> 当日下跌（涨幅 ≤ -0.1%）且成交量 > 前一日，即计为1个抛盘日。

| 条件 | 量化规则 |
|------|--------|----------------|
| 价格条件 | 收盘价 < 前一日收盘价 | 
| 成交量条件 | 当日成交量 > 前一日 | 
| 幅度条件 | 当日涨跌幅 ≤ **-0.1%**（避免平盘或微跌0.00%被误计） |

**常见问题** 

有人写的规则对收盘价在当天的位置做要求，但若某日下跌-0.3%、放量，但收盘价刚好在当日振幅的中部（例如开盘下跌、盘中反弹、收盘微跌），这条规则会判定“不是抛盘日”，但欧奈尔原意会算——因为**放量下跌本身就是机构抛售痕迹**。所以我的规则不对收盘价在当天的位置做要求。

### 2.2 特殊抛盘日（假阳线/滞涨）

**定义：**
> 当日收涨但涨幅 < 0.2%，成交量 > 前一日1.3倍，且日K线上影线长度 ≥ 实体长度 × 1.5（或上影线 ≥ 当日振幅一半），则计为1个抛盘日。

| 条件 | 量化规则（AND） |
|------|----------------|
| 收盘涨幅 | < 0.2% |
| 盘中最高涨幅 | ≥ 0.5%（确保有冲高动作） |
| 成交量 | > 前一日 × 1.3 |
| 上影线 | ≥ 实体 × 1.5 或 ≥ 振幅 × 0.5 |

四个条件必须同时满足，才计为1个特殊抛盘日。

**常见问题：**

“当日最高价涨幅 ≥ 0.5%”是为了：**排除开盘即横盘、全天无冲高动作的“假信号”**。

如果没有这个条件，一个全天在±0.1%内窄幅波动、收盘微涨0.15%、成交量略放大1.3倍、但上影线并不长的交易日，也可能被误判为“特殊抛盘日”。而欧奈尔定义这类抛盘日的核心是：**早盘或盘中曾有冲高（诱多），随后放量回落**，说明机构在上涨中悄悄派发。如果全天根本没有冲高，那就只是普通滞涨，不应计入抛盘日。

### 2.3 盘中反转抛盘日

**定义：**
> 当日收阴（收盘 < 开盘），成交量 > 前一日1.2倍，日K线存在明显上影线（上影线 ≥ 实体×1.5），且盘中最高曾上涨 ≥ 0.5%后回落。

| 条件 | 原规则 | **建议修改为** |
|------|--------|
| 价格条件 |  **收盘价 < 开盘价**（即收阴）即可，不限制跌幅大小 |
| 成交量 | > 前一日 × 1.2 | 
| K线形态 | **上影线 ≥ 实体 × 1.5** 且 **收盘价 ≤ (当日最高+最低)/2** |
| 额外条件 | **盘中最高价 ≥ 前一日收盘价 × 1.005**（确保确实出现过冲高） |

**常见问题：**

> 盘中反转抛盘日**可与标准抛盘日叠加吗？——不能，同一天只计1个抛盘日**，按优先级：反转日 > 特殊抛盘日 > 标准抛盘日。

## 3. 抛盘日的计数规则

### 3.1 基础计数规则

| 项目 | 规则 |
|--------|----------------|
| **幅度过滤** | 涨跌幅绝对值 < 0.05% 的“平盘日”不计为任何抛盘日 |
| **多重计数规则** | 若指数单日跌幅 ≥ **-1.5%** 且放量，计为 **2个抛盘日**（欧奈尔允许） |
| **确认日抵消机制** | 升势确认日：涨幅 ≥ 1.5% 且成交量 ≥ 前一日 → 可抵消 **1个** 抛盘日（仅抵消最早的那个） |
| **窗口滚动** | 严格滚动 25 个交易日，超过25天的抛盘日自动移出统计 |

### 3.2 权重系统（重抛日规则）

**权重是抛盘日计数规则的核心概念**，表示单个交易日在统计中的计数倍率：

| 权重值 | 含义 | 抛盘日类型 | 计数规则 |
|--------|------|------------|----------|
| **1** | 标准抛盘日 | 标准抛盘日、特殊抛盘日、盘中反转抛盘日 | 计为 **1** 个抛盘日 |
| **2** | 重抛盘日 | 标准抛盘日且跌幅≥-1.5%且放量 | 计为 **2** 个抛盘日 |
| **0** | 无抛盘日 | 平盘日或非抛盘日 | 不计入统计 |

**重抛盘日（权重=2）判定条件**：
1. 满足标准抛盘日所有条件（跌幅≤-0.1%且成交量>前一日）
2. **额外条件**：跌幅≥-1.5% 且 成交量放大
3. **计数效果**：该日按2个抛盘日计入加权总数

### 3.3 标准抛盘日成交量比率调整（重要更新）

**原规则**：成交量 > 前一日 × 1.0（只需放量）

**现规则**：成交量 > 前一日 × 1.05（需要放量5%）

**调整原因**：
- 1.0过于敏感，小幅成交量波动可能导致误判
- 1.05更贴近人工判断，要求明显放量
- 减少噪声，避免假信号

**参数化支持**：该阈值可在回测页面实时调整（默认值：1.05）


## 4. 抛盘日累积规则

### 4.1 加权抛盘日总数

市场状态判断基于**加权抛盘日总数**，而非原始数量：

```
加权总数 = ∑(每个抛盘日的权重)
其中：权重 = 1（标准/特殊/反转抛盘日）或 2（重抛盘日）
```

**示例计算**：
- 标准抛盘日：7天 × 权重1 = 7
- 重抛盘日：5天 × 权重2 = 10  
- 加权总数 = 7 + 10 = 17

### 4.2 市场状态判断

基于最近 **25个交易日** 的加权抛盘日总数：

| 市场状态 | 条件（加权总数） | 投资建议 |
|----------|------------------|----------|
| **正常状态** | 加权总数 < 5 | 正常选股，保持关注 |
| **承压状态** | 加权总数 ≥ 5 且 < 8 | 谨慎选股，控制仓位，密切关注市场变化 |
| **熊市状态** | 加权总数 ≥ 8 | 暂停选股，降低仓位，等待市场企稳 |

### 4.3 多指数扫描

**系统同时扫描两个主要指数**：
1. **中证全指（000985）** - 全市场代表
2. **沪深300（000300）** - 大盘股代表

**信号加强机制**：
- 如果两个指数同时出现相同信号，信号可信度更高
- 分别统计两个指数的市场状态，提供综合判断

### 4.4 参数化配置

所有阈值均可通过回测页面实时调整：
- 标准抛盘日阈值（默认：-0.1%）
- 标准抛盘日成交量比率（默认：1.05）
- 重抛盘日阈值（默认：-1.5%）
- 承压状态阈值（默认：5）
- 熊市状态阈值（默认：8）
- 分析窗口天数（默认：25）

### 4.5 透明化统计

系统显示所有抛盘日类型的详细统计，即使数量为零：
- 标准抛盘日数量
- 特殊抛盘日数量  
- 盘中反转抛盘日数量
- 重抛盘日数量
- 原始总数（权重未加权）
- 加权总数（权重加权后）

### 5.6 确认日抵消机制

**已全面启用**：
- ✅ 大盘扫描页面 (`market_scan.html`)
- ✅ 抛盘日摘要API (`/api/distribution/summary`)
- ✅ 参数化回测页面 (`backtest.html`)

**抵消规则**：
1. **按时间顺序抵消**：总是抵消窗口内日期最早的抛盘日
2. **权重处理**：一个确认日抵消1点权重
   - 标准抛盘日（权重1）→ 完全抵消（权重变为0）
   - 重抛盘日（权重2）→ 部分抵消（权重从2变为1）
3. **完全抵消**：权重减为0后，该交易日不再计入抛盘日统计
4. **透明统计**：详细抛盘日列表显示抵消后的实际权重

**技术实现**：
- `DistributionWindow.apply_confirmation_offset()` - 核心抵消逻辑
- 严格的异常处理和类型安全检查
- 与 `MultiIndexScanner` 深度集成，确保多指数分析一致性
